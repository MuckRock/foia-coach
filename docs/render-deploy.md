# Deploying to Render

## Prerequisites

- A [Render](https://render.com) account
- The repo connected to Render (via GitHub)
- Recompiled `api/requirements.txt` with `whitenoise[brotli]` included

## Quick deploy

1. In the Render dashboard, click **New** > **Blueprint**
2. Select the repo and point it to `render.yml`
3. Render will create three resources:
   - **foia-coach-db** — PostgreSQL database
   - **foia-coach-api** — Django API (Python)
   - **foia-coach** — SvelteKit frontend (Node)

## Environment variables to set manually

Several env vars use `sync: false` in `render.yml`, meaning they must be set in the Render dashboard after the blueprint deploys.

### API service (`foia-coach-api`)

| Variable              | Description                                                |
| --------------------- | ---------------------------------------------------------- |
| `MUCKROCK_API_URL`    | URL for the MuckRock API (e.g. `https://www.muckrock.com`) |
| `MUCKROCK_API_TOKEN`  | Auth token for the MuckRock API                            |
| `CORS_ALLOWED_ORIGIN` | Frontend URL (e.g. `https://foia-coach.onrender.com`)      |
| `GEMINI_API_KEY`      | Google Gemini API key                                      |
| `OPENAI_API_KEY`      | OpenAI API key                                             |

To enable real LLM API calls, also update:

| Variable                  | Change to            |
| ------------------------- | -------------------- |
| `RAG_PROVIDER`            | `openai` or `gemini` |
| `OPENAI_REAL_API_ENABLED` | `true`               |
| `GEMINI_REAL_API_ENABLED` | `true`               |

### Frontend service (`foia-coach`)

| Variable         | Description                                                  |
| ---------------- | ------------------------------------------------------------ |
| `PUBLIC_API_URL` | API service URL (e.g. `https://foia-coach-api.onrender.com`) |

This is a static env var baked in at build time. After changing it, trigger a manual deploy of the frontend service.

## Auto-configured variables

These are set by `render.yml` and don't need manual changes:

| Variable                 | Value                    | Notes                                    |
| ------------------------ | ------------------------ | ---------------------------------------- |
| `DATABASE_URL`           | (from Render DB)         | Connection string injected automatically |
| `SECRET_KEY`             | (generated)              | Random value generated by Render         |
| `DJANGO_SETTINGS_MODULE` | `config.settings.render` | Uses Render-specific settings            |
| `WEB_CONCURRENCY`        | `4`                      | Gunicorn worker count                    |

## What happens at build time

### API (`api/build.sh`)

1. `pip install -r requirements.txt`
2. `python manage.py collectstatic --no-input` — gathers static files (served by WhiteNoise)
3. `python manage.py migrate` — runs database migrations

### Frontend

1. `npm install`
2. `npm run build` — builds the SvelteKit app with `adapter-node`

## Troubleshooting

- **CORS errors**: Make sure `CORS_ALLOWED_ORIGIN` on the API service matches the frontend URL exactly (including `https://`).
- **Static files 404**: Verify `whitenoise[brotli]` is in `requirements.txt`. Run `uv pip compile requirements.in -o requirements.txt` and redeploy.
- **Frontend shows wrong API URL**: `PUBLIC_API_URL` is baked in at build time. After changing it, you need to trigger a new build.
- **Database connection errors**: Check that the `foia-coach-db` database is running and the `DATABASE_URL` env var is linked to it in the API service settings.
